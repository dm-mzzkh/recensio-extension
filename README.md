# Recensio

Расширение для Firefox (MV3), которое сохраняет видео с YouTube вместе с оценками, тегами и заметками. Вся информация хранится локально в IndexedDB через Dexie; никаких внешних сервисов.

## Что можно делать с видео YouTube

Расширение подключается к страницам `https://www.youtube.com/watch*` и добавляет три кнопки рядом с заголовком ролика, плюс открывает редактор и поддерживает горячие клавиши.

### ★ Recensio — оценка, рецензия, теги

Кнопка `★ Recensio` рядом с заголовком открывает оверлей-редактор поверх страницы YouTube. В нём для текущего видео можно:

- **поставить оценку 1–10** (поле `rating`);
- **написать рецензию/заметку** в свободной форме (поле `review`);
- **навесить теги** — теги нормализуются (общие для всей библиотеки), при вводе показываются подсказки из уже существующих;
- видео автоматически добавляется в библиотеку при первом сохранении (`videoId`, `addedAt` и т.д. берутся из URL).

Закрыть оверлей — клик мимо рамки, кнопка `×` или клавиша `Esc`.

### 📷 Скриншот текущего кадра

Кнопка `📷` (или горячая клавиша **Alt+Shift+S**) снимает текущий кадр плеера:

- кадр рисуется в `<canvas>` и кодируется в JPEG (качество 0.92);
- скриншот привязывается к видео и к таймкоду `currentTime`;
- сохраняется локально через background-скрипт;
- если воспроизведение идёт, ждётся ближайший `requestVideoFrameCallback`, чтобы получить ровный кадр без разрыва;
- если ролик защищён DRM (`video.mediaKeys != null`) или canvas «отравлен», показывается ошибка `Видео защищено DRM — снимок невозможен`.

Снизу появляется тост с таймкодом сохранённого кадра, например `📷 12:34 сохранён`.

### ✂ Клип: вырезать фрагмент видео

Кнопка `✂` (или **Alt+Shift+C**) ставит метки начала и конца клипа прямо во время просмотра. Логика двух нажатий:

1. **Первое нажатие** — фиксирует `startSec = currentTime`. Кнопка становится жёлтой: `✂ 1:23 → …`. Под заголовком разворачивается панель с текущим диапазоном.
2. **Второе нажатие** — фиксирует `endSec = currentTime` (должен быть позже начала, иначе ошибка `Конец должен быть позже …`). Кнопка становится зелёной: `✂ 1:23 – 2:45`. В панели появляются кнопки **↓ Сохранить** и **✕ Отменить**.
3. **Третье+ нажатие** (когда уже обе метки) — переставляет конец клипа на новый таймкод, чтобы можно было подправить без сброса.

При сохранении клипа:

- запись о клипе создаётся в БД (`videoId`, `startSec`, `endSec`, статус/стадия);
- в буфер обмена копируется готовая команда `yt-dlp`, например:
  ```
  yt-dlp -f "bv*+ba/b" --download-sections "*00:01:23.00-00:02:45.00" "https://www.youtube.com/watch?v=…"
  ```
- background-процесс параллельно запускает фоновую запись клипа через native messaging host (`recensio:start-recording`), чтобы файл сохранился на диск без участия пользователя;
- показывается тост вида `✂ 1:23–2:45 (82.0 c) · yt-dlp скопирован · идёт фоновая запись…`.

Отменить незавершённый клип — кнопка `✕ Отменить`, либо `Esc` (если не открыт оверлей редактора). При SPA-навигации YouTube (`yt-navigate-finish`) незавершённые метки сбрасываются.

### Горячие клавиши

| Сочетание | Действие |
| --- | --- |
| **Alt+Shift+S** | Снять скриншот текущего кадра |
| **Alt+Shift+C** | Поставить метку клипа (начало → конец) |
| **Esc** | Закрыть оверлей редактора или отменить незавершённый клип |

### Как это видится в библиотеке

Сохранённые видео, скриншоты и клипы доступны через popup расширения (кнопка в тулбаре) и страницу-библиотеку. Поиск по библиотеке поддерживает фильтры:

- по тегам (AND-совпадение по списку имён);
- по диапазону оценок (`ratingMin` / `ratingMax`).

Есть отдельная страница-граф для визуализации связей между видео и тегами.

### Бэкап и импорт

На странице-библиотеке есть кнопки рядом с заголовком и два тумблера под ними.

- **↓ Бэкап** — выгружает всю базу в `~/Downloads/recensio-backup/`: сжатый `manifest.json.gz` со всеми таблицами (видео, теги, системные теги, скриншоты, клипы — метаданные) плюс сами блобы (`blobs/sc-*.jpg`, `blobs/cl-*.mp4` и т.п.). Выгрузка **инкрементальная**: локальный реестр (`backupLedger`) помнит, какие блобы уже выгружены, и повторный бэкап шлёт только новые файлы. Манифест перезаписывается целиком каждый раз. Требует разрешения `downloads`.
- **↑ Импорт** — выбирается папка `recensio-backup` целиком (через `webkitdirectory`). **Импорт затирает текущую базу** и заменяет её содержимым манифеста (с подтверждением). Схема бэкапа должна совпадать с текущей версией расширения — более старые/новые снапшоты отклоняются (миграция при импорте не реализована). Отсутствующие на диске блобы пропускаются, метаданные при этом сохраняются.
- **⤵ Merge** — объединяет выбранную папку с текущей базой, **ничего не удаляя**. Видео сводятся по `videoId` по принципу last-write-wins (побеждает запись с большим `updatedAt`); теги — по имени; скриншоты, клипы, видео-теги и системные теги дедуплицируются по естественным ключам (`[videoId+createdAt]`, `[videoId+startSec]` и т.п.), так что повторный merge того же бэкапа не плодит дубли. Теги при merge объединяются (union), а не затираются.

Два тумблера (по умолчанию включены, состояние в `storage.local`):

- **Авто-бэкап каждые 5 мин** — фоновый процесс (через `browser.alarms`) раз в 5 минут повторяет экспорт. Чтобы не засорять историю загрузок одинаковыми манифестами, прогон **пропускается, если данные не менялись** с прошлого раза (сравнивается дешёвая сигнатура: счётчики таблиц + макс. таймстемпы). Пустая база не экспортируется — чтобы свежий профиль не затёр уже лежащий в папке бэкап.
- **Предлагать восстановление при пустой базе** — если при открытии библиотеки база пуста, показывается баннер с кнопкой восстановления (открывает тот же folder-picker и делает импорт-замену). Расширение не может читать файлы с диска само (Downloads API только пишет), поэтому восстановление — в один клик пользователя, а не полностью молчаливое.

## Требования и установка

- Firefox 115+ (MV3, Gecko-only).
- Для фоновой записи клипов — установленный native-host (см. `native-host/README.md`) и `yt-dlp` в `PATH`.
- Сборка: `npm install && npm run build`. Запуск в браузере: `npm start` (через `web-ext`).
- Линт: `npm run lint` (типы) и `npm run lint:ext` (web-ext).
